Install OKChroma manually

A full accessibility-solved color system from one brand color — written into your file as Figma variables. The Figma Community listing is in review; this page installs the exact same plugin manually. It takes about a minute.

⬇  Download OKChroma

Rebuilt automatically from the latest engine — this link is always current.

You'll need the Figma desktop app — the browser version can't install plugins this way. (Free from figma.com/downloads.)
  1. Unzip the download.You'll get a folder called okchroma-plugin. Put it somewhere it can stay — Figma reads it from this location every time the plugin runs.
  2. Open any design file in Figma desktop.A new empty file works fine.
  3. Import the plugin.Menu (the Figma logo, top-left) → Plugins → Development → Import plugin from manifest… — then pick manifest.json inside the unzipped folder.
  4. Run it.From now on it lives under Plugins → Development → OKChroma. Name a brand, pick a color, hit Apply to Figma, then open the Variables panel.

Good to know

• The plugin installs just for you, on this computer — teammates each do the four steps once. • To update later: download the zip again, replace the old folder's contents, done — no re-import needed if the folder stays in the same place. • It never touches the network and stores nothing outside your file. • When the Community version is approved, install that one instead and delete the development copy.

OKChroma Extended — for Figma Enterprise

• A second plugin built on Figma's extended collections: one theme base (modes wcag · wcag-dark · apca · apca-dark) plus one extension per brand, overriding only what differs. Extended collections are currently an Enterprise-plan feature — on other plans the plugin says so and writes nothing. • Not on Community yet — install it the same way: Download OKChroma Extended, same four steps, with a folder called okchroma-extended.
